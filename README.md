# 🤖 RoboReviews: AI-Powered Product Review Aggregator

## Project Overview

RoboReviews is an end-to-end, AI-powered pipeline designed to automate the analysis of thousands of Amazon consumer reviews. The tool intelligently clusters products, ranks them based on user sentiment, and generates final, blog-post style reviews for each category. For every product category analyzed, RoboReviews recommends the **Top 3 products** to buy and highlights the **Worst product** to avoid, saving consumers countless hours of manual research.

## Project Architecture (The 3 Models)

The system is built on a robust three-step methodology, combining Natural Language Processing, Machine Learning, and Generative AI:

### 1. Data Preprocessing & Sentiment Analysis

* **Data Ingestion:** Merged 3 distinct Kaggle datasets containing raw Amazon product reviews.

* **Sentiment Mapping:** Standardized the labels by mapping star ratings to sentiments: `1-2 stars = Negative`, `3 stars = Neutral`, and `4-5 stars = Positive`.

* **Classification:** Utilized the Hugging Face transformer model `cardiffnlp/twitter-roberta-base-sentiment` to perform granular sentiment classification directly on the review text.

### 2. Clustering

* **Vectorization:** Transformed product descriptions and reviews using a TF-IDF matrix. To improve semantic accuracy, the matrix was aggressively cleaned (e.g., removing ubiquitous, non-informative words like 'Amazon').

* **Rule-Based Overrides:** Applied hardcoded rule-based overrides to ensure edge cases were categorized logically (for instance, explicitly forcing cables and adapters into an "Accessories" category).

* **K-Means Application:** Executed the `KMeans` algorithm to group the filtered product data into **5 distinct meta-categories**.

### 3. Generative AI Summarization

* **Ranking:** Sorted products within each generated meta-category to mathematically extract the Top 3 best-received products and the single Worst-rated product.

* **LLM Integration:** Leveraged OpenAI's `gpt-4o-mini` model using a highly strict, structured prompt.

* **Content Generation:** Synthesized a 300-400 word blog post for each category. The AI explicitly compares the products, highlights real user complaints, and is rigidly constrained to avoid hallucinations.

## Directory Structure

```
roboreviews/
├── main.ipynb                          # The main Jupyter Notebook containing the full pipeline code
├── .env                                # Environment variables (must contain your OpenAI API Key)
├── blog_posts_generated.txt            # Final text output containing Meta-Categories, TOP 3 PRODUCTS, WORST PRODUCT, and the LLM-generated blog articles
└── data/                               # Directory containing all datasets
    ├── 1429_1.csv                      # Raw Kaggle dataset (Example)
    ├── [other_raw_kaggle_files.csv]    # Additional raw dataset files
    └── processed_reviews_with_sentiment.csv # The combined, cleaned dataset generated by the script
```

* **`main.ipynb`**: Your primary execution file.

* **`data/`**: Ensure your raw datasets are placed here before running the pipeline. The script will also output the intermediate sentiment data to this folder.

* **`blog_posts_generated.txt`**: The final destination for the LLM's consumer reports.

## Installation & Setup

1. **Clone the repository** (if applicable) and navigate to the project directory.

2. **Install the required libraries:**
   You will need several Python packages for data manipulation, machine learning, and API interactions. Run the following command:

   ```bash
   pip install pandas numpy scikit-learn transformers openai wordcloud python-dotenv
   ```

3. **Configure the Data Folder:**
   Download the required Amazon review datasets from Kaggle and place the `.csv` files (e.g., `1429_1.csv`) directly inside the `data/` folder.

4. **Set Up the Environment Variables:**
   Create a new file named `.env` in the root directory of the project and add your OpenAI API key:

   ```env
   OPENAI_API_KEY=your_openai_api_key_here
   ```

## How to Run

1. Open the Jupyter Notebook:

   ```bash
   jupyter notebook main.ipynb
   ```

2. **Execute Sequentially:** Run the notebook cells from top to bottom. The pipeline will automatically:

   * Load and clean the CSV files from the `data/` folder.

   * Run the Hugging Face sentiment analysis.

   * Cluster the products via KMeans.

   * Call the OpenAI API to generate the final summaries.

3. **View the Results:**
   * The generated content will print directly within the notebook's output cells.
   * A permanent copy of the output will be saved automatically to `blog_posts_generated.txt`, giving you a clean, copy-paste-ready blog post for each category.
